VidPlay

Release Notes & Changelog

v2.1.0 Latest
July 2026
New Features
  • Smart Video Resume: When re-opening a previously watched video, a sleek popup appears showing your last position with a "Continue" button. Auto-hides after 10 seconds. Triggers only after the video is fully loaded.
  • Video Enhancer (AI Profiles): Added offline AI-style color profiles — Cinematic, HDR Boost, Vivid, Dark Detail, and more — with real-time color matrix overlays on the player.
  • Separate Audio & Subtitle Tracks: The track selector now splits Audio and Subtitle into dedicated panels with proper icon separation.
  • Buffering Performance Overlay: When loading large/high-quality files, a real-time processing popup shows encoding, decoding, and codec metadata.
  • Smart Playlists Redesign: Completely rebuilt the Playlists page with a professional layout — stats row, featured hero card, 2-column grid, themed accent colors, multi-thumbnail mosaics, and press animations.
Improvements
  • Playlist Card UI: New multi-thumbnail mosaic (2×2 grid), accent color icon badge, play-all FAB, total duration display, and spring press animation.
  • Playlist Detail View: Themed header banner with playlist icon, video count, description, and a "Play" button when entering a playlist.
  • Player Top Toolbar: Added Magic Wand icon for Video Enhancer in both collapsed and expanded toolbar states.
  • Transparent Backgrounds: Applied radial gradient dark-transparent backgrounds across all screens for a consistent glassmorphism look.
  • Play Icon Size: Reduced the play icon size in the pause-state ad popup to avoid visual clutter.
Bug Fixes
  • Pinch-to-Zoom Gesture Conflict: Fixed a critical bug where zooming the video with two fingers would hijack single-finger swipe gestures (volume, brightness, seek). Zoom now exclusively responds to multi-touch.
  • Panning Speed Bug: Removed a broken scale multiplier that caused the zoomed video to move too fast when panning.
  • Resume Button Not Working: Fixed the "Continue" button starting from 0:00 by deferring the seek until the player reaches STATE_READY.
  • Subtitle Rendering: Fixed subtitles not displaying by explicitly enabling subtitleView visibility and applying drop-shadow styling.
v1.1.0
June 2026
New Features
  • Custom Crash Reporter: Added a global crash interceptor. If the app fails, you are greeted with a recovery screen offering 1-click GitHub reporting, WhatsApp log sharing, and clipboard copy functionality.
  • A-B Repeat Mode: You can now set point A and point B on the timeline to infinitely loop a specific segment of video.
  • Permanent Bookmarks/Chapters: Integrated Room Database to permanently save timestamps and chapters for local videos.
  • Video Adjustments Panel: Added robust video modifiers including aspect ratio resize (Fit, Fill, Zoom), mirroring, flipping, and 360-degree rotation.
  • Auto Playlist Progression: Hitting the "Next" or "Prev" button now intelligently jumps to the neighboring video file in the current folder directory.
Improvements
  • Compact Player Controls: Completely overhauled the bottom controller layout. Merged the timeline slider, duration, and action icons (Play/Pause, Next, Prev) into a single space-efficient row.
  • Icon Uniformity: Shrunk the Play/Pause icon to precisely match the size (36dp) of the surrounding Prev/Next controls for a sleeker aesthetic.
  • Auto-Hide UI: The playback controls and top toolbar now automatically disappear after 4 seconds of playback to preserve immersion.
  • Glassmorphism Upgrade: Replaced flat backgrounds with 80% opaque blurred backgrounds across menus, sheets, and dialogs.
  • Android 16 Support: Target SDK bumped to 36 with full Edge-to-Edge compliance.
Bug Fixes
  • Slider Crash: Resolved a critical IllegalArgumentException crash caused when the video scrubber attempted to reach out-of-bounds timestamps.
  • Status Bar Artifacts: Fixed an issue where the app title "HOAVideo Player" was appearing over the video. App now utilizes complete fullscreen immersive mode.
  • PiP Transition: Fixed a bug where returning from Picture-in-Picture mode occasionally distorted the aspect ratio.
